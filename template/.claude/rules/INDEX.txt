## Customization Guide

Files to customize for your project:
- `projects/registry.md` - Replace example projects with yours
- `technical/patterns.md` - Update port number and format commands
- `quality/standards.md` - Set your health check URLs

Files that work as-is (no customization needed):
- All other files are universal best practices

---

## How Rules Work

1. **Auto-discovered**: Claude Code finds `.claude/rules/*.md` automatically
2. **Recursive**: Subdirectories are searched
3. **Priority**: Project rules override user rules (`~/.claude/rules/`)
4. **Path-specific**: YAML frontmatter can target specific file paths

---

## Adding New Rules

1. Create `.md` file in appropriate subdirectory
2. Add YAML frontmatter for path-specific rules (optional)
3. Update this INDEX file
4. Keep rules focused (one domain per file)
